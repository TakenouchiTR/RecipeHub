Use-Case Scenario: Edit an Ingredient
Users: Chef
Context: User is viewing the ingredients screen
Primary Flow:
    1: User selects the "edit" option for an ingredient
    2: User is prompted to modify the quantity of the ingredient
    3: User is prompted to modify the unit of measurement for the ingredient
    4: User confirms and submits the edits to the ingredient
    5: The ingredients screen is updated to reflect the edited ingredient
Alternative Flows:
    a1:
        1: The user does not have any ingredients added
        2: The user is unable to edit ingredients until at least one ingredient is added
    a2:
        1: The user enters a negative quantity
        2: The user is informed that they cannot enter a negative quantity
        3: The user is returned to step 5
    a2-3:
        1: The user closes the prompt
        2: The user is returned to the ingredients screen eithout any changes
    a4:
        1: The application is unable to connect to the server
        2: The user is informed of the connection issues
        3: The user is returned to step 2
