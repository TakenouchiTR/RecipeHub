Use-Case Scenario: Create an Account
Users: Chef
Context: User launches the website or desktop application and wants to create a new account
Primary Flow:
    1: User selects the option to create an account
    2: User enters their desired username
    3: System verifies that the username is unique
    4: User enters their password
    5: User verifies their password
    6: User enters their first name
    7: User enters their last name
    8: User enters their email address
    9: User selects their account type
    10: User submits their information and an account is created
    11: User is returned to the login screen
Alternate Flows:
    a2: 
        1: Username does not meet requirements (Length, legal characters)
        2: The user is informed of the error in their username
        3: The user is returned to step 2
    a2-9:
        1: The user cancels account registration
        2: The user is returned to the login screen
    a3: 
        1: Username is not unique
        2: the user is informed that their username already exists
        3: The user is returned to step 2
    a4: 
        1: Password does not meet requirements (Length, required characters)
        2: The user is informed that their password does not meet requirements
        3: The user is returned to step 4
    a5: 
        1: Password and verify password fields do not match
        2: The user is informed their their password and retyped password do not match
        3: The user is returned to step 5
    a6: 
        1: First name does not meet requirements (Length, legal characters)
        2: The user is informed of the error in their first name
        3: The user is returned to step 6
    a7: 
        1: Last name does not meet requirements (Length, legal characters)
        2: The user is informed of the error in their first name
        3: The user is returned to step 7
    a8: 
        1: Email does not meet requirements (Format, legal characters)
        2: The user is informed of the error in their first name
        3: The user is returned to step 8
    a10:
        1: The user has unresolved issues with their account registration
        2: The user is prompted to fix their issues
        3: The user is returned to step 10
